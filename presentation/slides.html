Solar geometry

Real sun path

Declination, altitude, and azimuth calculated from day-of-year + hour. Ray-AABB shadow casting against every structure. No external APIs.

Methodology
How I used Claude Code

Plan → execute → iterate.

  • Plan mode first. For anything non-trivial I asked Claude to draft an implementation plan, then approved it before any file was touched.
  • Small, scoped prompts. One feature per prompt. Each prompt read like a product spec, not a code request.
  • Commit after every feature. 16 commits in 12 hours — each one buildable and revertable.
  • Auto-memory for corrections. Mistakes (like the localStorage incident) were written to persistent memory so they wouldn't repeat.

Sun heatmap

Multi-layer exposure maps

Multiple heatmap instances with independent center, size, and mode. Shows peak sun hours, direct exposure, or total daylight across the yard.

Smart alerts

"Your tomatoes need more sun."

Per-plant peak-sun-hour thresholds. Click an alert to jump to the bed and auto-flash the sun heatmap. Dismiss until the bed is moved.

Lessons
The biggest mistake

I lost every saved config.
Once.

During a "fresh-load" test, Claude ran localStorage.clear() and nuked three saved garden layouts I'd been iterating on for an hour.

The fix: a persistent memory rule — never clear localStorage without explicit permission. Written once, respected every session after.

Auto-memory makes mistakes teach.

Takeaways
What I learned

The lesson isn't "AI is fast."

It's that planning is the work now. The best hours of this build were the ones where I sat with a plan before any code got written. Claude executed; I architected.

  • Spec like a PM, not a coder. Describe the user-visible behavior, let Claude pick the implementation.
  • Commit relentlessly. Every feature is a checkpoint. Reverts are free.
  • Let mistakes become rules. Auto-memory turns one-time incidents into permanent guardrails.
  • Trust but verify. Read every diff. Run the dev server. Open the browser.
